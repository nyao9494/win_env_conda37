    Force use of MSVC's stdint.h when available
    
    If the MSVC version used has `stdint.h` (namely VS 2015+), then force

    encountered with Python 3.6 and allow for a more proper build on Python
    3.5 when using MSVC.

Author: John Kirkham <kirkhamj@janelia.hhmi.org>
Date:   Mon Mar 6 19:47:19 2017 -0500

    Skip Pythons beyond 3.5 as they are unsupported.
    
    Currently am seeing build issues with Python 3.6 on Windows. Given these
    failures, the fact that the source is heavily patched, and upstream has
    basically stopped maintaining the library, it seems safer to explicitly
    disable Python 3.6 or beyond. Otherwise we may be stuck maintaining our
    own fork of PyCrypto. Instead it makes more sense to encourage
    developers to use PyCryptodome in its place.
